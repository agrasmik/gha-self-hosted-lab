## Основные понятия и термины

*Ссылка на источник*: https://docs.github.com/en/actions/get-started/understand-github-actions

**Workflow** - Рабочий процесс. 
По сути группировка задач, для достижения одной или нескольких целей. Для каждого репозитория может быть несколько Workflow, которые запускаются при разных событиях. Все Workflow находятся по пути `.github/workflows`.
Workflow могут ссылаться друг на друга.
При возникновении какого-либо события в репозитории и наличия этой директории, GitHub пробегает по всем файлам с Workflow, сравнивает произошедшее события с событиями в написанных файлах, строит общий план заданий и запускает их на выполнение.

**Events** - События.
GitHub Actions использует событийную модель. Это значит, что он отслеживает все события, происходящие с репозиторием. Это позволяет очень гибко настраивать пайплайны - от запуска уведомлений в общий чат о новых Issue, до действительно сложных комбайнов с запуском всех возможных этапов тестирования на различных ветках/стадиях разработки, сборки проекта и сложного деплоя в различные среды окружения. 

**Jobs** - Задачи.
Задача - это набор "шагов", которые необходимо выполнить. По сути это скрипт с расширенными возможностями. Каждая задача запускается в изолированной среде и представляет из себя последовательность команд. Кроме того, задачи могут содержать и Actions. 
По-умолчанию все задачи одного Workflow запускаются параллельно. Но это поведение можно исправить. Ещё написано про какие-то ***матрицы***. На сколько я понял - из задачи можно сделать некоторое подобие функции и запускать её с различными параметрами. 

**Actions** - Прямой перевод *"Действия"* сюда подходит не очень хорошо, т.к. не совсем верно передаёт смысл. Больше подойдёт *"Сценарии"* или даже "Блоки". Но лучше использовать исконное *"Экшонс"*.
Это более стандартизированные и чаще используемые функции. Могут включать в себя несколько Задач или заранее подготовленный скрипт. Предназначены для сокращения кода (DRY). Существуют предоставленные от GitHub заготовленные Actions (например Actions для клонирования репозитория `actions/checkout@v5`). Можно и самому писать свои Actions или использовать написанные другими людьми (размещаются на маркетлпейсе от GitHub).

**Runners** - Раннеры. Прямого или перевода вообще не найти.
Раннеры предназначены для запуска задач, описанных в Workflow. У GitHub есть собственные (публичные) Раннеры. Их можно использовать либо с ограничениями либо платно, либо и то и другое.
Так же можно добавлять собственные Раннеры в свои проекты (эти мы тоже тут будем заниматься). На сколько я понял, для каждой задачи будет запускаться собственный докер-контейнер на раннере, который подходит под критерии задачи/Workflow. Документация пишет, что для каждого Workflow запускается собственная чиста виртуалка, и на ней уже выполняются Задачи. Будем тестировать.
